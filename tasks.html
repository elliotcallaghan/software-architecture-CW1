Homework 1: Tasks for ejc62

Your personal task is to develop a system of movies and details about their actors.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2103.hw1.domain.Actor. With the following private fields:

  1. String surname
  2. String firstName
  3. String language
  4. int age
Generate necessary getters and setters. [7 marks]

Create class co2103.hw1.domain.Movie. With the following private fields:

  1. int id
  2. String title
  3. String city
  4. List<Actor> actors
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2103.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Movie> movies in class co2103.hw1.Hw1Application. Initialize the list inside class co2103.hw1.Hw1Application to contain one movie with two actors. [4 marks]

Create controller co2103.hw1.controller.MovieController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/movies" should show view movies/list. [2 marks]
    • Add the list of all movies from co2103.hw1.Hw1Application to the model as attribute movies. [2 marks]
    • Create the corresponding JSP to list the title and city of each movie followed by a link to URL "/actors?movie={id of movie}", e.g., "/actors?movie=3". [4 marks]
  2. Requests for URL "/newMovie" should show view movies/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="movie" and action="/addMovie" and inputs for all fields of class Movie except for actors. [5 marks]
  3. POST requests for URL "/addMovie" should add a new movie to the list of all movies in co2103.hw1.Hw1Application and redirect to /movies. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2103.hw1.controller.ActorController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/actors" should show view actors/list. [2 marks]
    • Based on request parameter movie determine the corresponding movie from co2103.hw1.Hw1Application. [2 marks]
    • Add the list of all actors of that movie to the model as attribute actors. [2 marks]
    • Create the corresponding JSP for the view to list the surname, firstName, and language of each actor. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newActor?movie={id of movie whose actors are listed}", e.g., "/newActor?movie=3". [4 marks]
  2. Requests for URL "/newActor" should show view actors/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="actor" and action="/addActor" and inputs for all fields of class Actor. [6 marks]
    • Also add the request parameter movie with the ID of the current movie. [4 marks]
  3. POST requests for URL "/addActor" should add a new actor to the list of actors of the movie identified by request parameter movie and then redirect to /movies. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Movie and make it available in controller MovieController. [4 marks]

When adding a new movie validate that

  1. The value of field id is unique within the list Hw1Application.movies. [3 marks]
  2. The value of field title is not empty. [2 marks]
  3. The value of field city is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Actor and make it available in controller ActorController. [4 marks]

When adding a new actor validate that

  1. The value of field surname is not empty. [2 marks]
  2. The value of field firstName is not empty. [2 marks]
  3. The value of field language is either "English", "German", or "Spanish". [3 marks]
  4. The value of field age is between (including) 1 and 120. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
